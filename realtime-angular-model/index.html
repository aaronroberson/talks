Known Issues

Javascript is asynchronous and stuff

Automatic promise unwrapping is out; good.
  • Implement unwrap on your base model, where it belongs
  • Return the model now, populate it whenever
Unit.prototype.$unwrap = function(futureUnitData) {
  var self = this;

  this.$futureUnitData = futureUnitData;
  this.$futureUnitData.then(function(data) {
    if (!self.$valid(data)) return new Error('Trouble in Paradise');
    Unit.$timeout(function() { _.extend(self, data); });
  });
};

Parasitic combination inheritance

var createObject = Object.create;

if (!_.isFunction(createObject)) {
  createObject = function(obj) {
    function F() {}

    F.prototype = obj;
    return new F();
  };
}

function inheritPrototype(SubClass, SuperClass) {
  var superCopy = createObject(SuperClass.prototype);
  superCopy.constructor = SubClass;

  SubClass.prototype = superCopy;
}

Events and context are required to maintain a local representation of your remote data

Minimum Required Events

  • Set: An existing value has changed
  • Add: A new property has been added
  • Remove: A property has been removed

Conclusion

Have I found the perfect model?
Probably not.

Is implementing your own model, worth the effort? Holy, yes.

Real-time makes for much improved user experiences, and best of all: The model is still awesome, It just needs to learn "a few new tricks"

You'll know you're getting it right when:

  • Your model is easy to unit & intergration test
  • Your model is easy to extend
  • Your model is portable
  • Your controllers & views are thin
